HTML 5

html标签的lang属性

不管是写了en还是zh-cn，里面都可以出现英文和中文，但是浏览器会在你指定en的时候以为你这是个英文网页，谷歌还会提示你是否要翻译此网页

html5的兼容处理

ie678不认识html5的新标签

解决方法：
可以通过js的document.createElement(“footer”),由于这样创建的标签后虽然浏览器能识别这个标签，但是会渲染成行内样式，需要加个display：block的属性。

很明显上述方法能行但是过于繁琐，可以使用别人写好了的轮子htmlshiv.js放在html的头部，这样那些不支持h5的浏览器就能识别h5标签了。

如果浏览器本身就支持h5标签，那么就不需要htmlshiv.js加载，以免造成性能损失，这里引出了一个比较重要的方法

css骇客 条件注释

<!--[if lte IE 6]>
    //do sth
<![endif]-->

这种注释是ie独有的注释，只有ie自己能识别到
可以把script标签方法注释体里面，符合条件才会运行注释里面的代码

  • lte <=
  • lt <
  • gt >
  • gte >=
    emmty语法：cc:ie6 ,(只能是6)

表单校验

invalid 表单校验失败时触发的事件，需要提交
inputElement.setlustomValidity(“xxx不对”);
require 非空验证
pattern=”正则”

html:

<input type="text" require pattern="\d+" id="input1">

script:

$("#input1").on("invalid",function(){
    $("#input1")[0].setlustomValidity("不是全数字");
    })

其他表单属性

multiple 多选 应用于file和selected 等
